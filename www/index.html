Introduction

FMIGo is a set of Free Software tools aimed at the Functional Mockup Interface (FMI) standard. FMI is a standard for packaging simulations into .zip files called Functional Mockup Units (FMUs). Each FMU contains an XML file describing the system contained within, and a set of binaries, data and possibly source code. What FMIGo does is provide a backend for running such FMUs, across one or more computers, using TCP/IP (ZeroMQ) or MPI for communication.

Why FMI?

One of the reasons FMI exists is so different simulation authoring tools can output a single standardized format. For example MSC ADAMS, Simulink and AGX Dynamics all support FMI output. This means they can all be executed by tools like PyFMI, DACCOSIM or FMIGo.

It is also possible to create FMUs outside the tools mentioned above, using for example QTronic's FMU SDK, which is licensed under the BSD license.

Why FMIGo?

FMIGo

  • Networked architecture allows connecting simulations that must run on separate physical machines due to licensing constraints
  • Ability to solve algebraic loops in initialization mode, if linked with the GNU Scientific Library (GSL) (see license)

Limitations

FMIGo is not for everything, so here we will list some limitations of the system:

  • The overhead between steps (message packing, communication, distributing values) is 54 µs on an Intel® Core™ i7-860 processor with 8 threads running at 2.8 GHz, on a test with 7 FMUs using MPI, not using kinematic coupling. This may be an issue for systems that need to run at 1 kHz or faster, such as hardware-in-the-loop (HIL) robotics systems. How?

In order for FMIGo to be of much use, you must pick some method of coupling your simulations. For physical systems FMIGo provides the SPOOK solver by Claude Lacoursière. Another option is to use the NEPCE method developed by Edo Drenth.

This site will only go over the gist of how to use the programs. Extensive invocation details are available in the manual.

License

FMIGo itself is licensed under the MIT license. GNU Scientific Library version (GSL) is licensed under the GNU General Public License version 3 (GPLv3), and is required for FMIGo to be able to solve algebraic loops during initialization. The user therefore has the choice of two license options: enable loop solving (GPL, default) or disable loop solving (MIT). To build without GPL, you must give cmake the option -DUSE_GPL=OFF.

For future version we may consider other license options. In order to guarantee that improvements to FMIGo are never locked away behind a cloud we may opt for something that fills the gap between the GPL and the GNU Affero General Public License (AGPL).
